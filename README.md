## DPG comparison вЂ” which to use

| | CREDEBL | INJI |
|--|---------|------|
| **Architecture** | Microservices (20 containers) | Modular stack (7 containers) |
| **Auth** | Keycloak | eSignet (MOSIP's native OIDC) |
| **Credential formats** | SD-JWT VC, AnonCreds | SD-JWT VC, W3C VCDM 1.1/2.0 |
| **Holder wallet** | External wallet (OID4VCI-compatible) | Inji Web included + Inji Mobile |
| **RAM usage** | ~4.5GB | ~3.5GB |
| **Best for** | Countries with existing wallet | Countries needing full stack with wallet |
| **Day 5 OIDC swap** | Keycloak в†’ country OIDC | eSignet в†’ country OIDC |

Select the DPG on Day 3 jointly with the country. Both are pre-built and ready.

## Quick start

### VPS setup (first time only)
```bash
sudo bash scripts/setup-vps.sh
```

### CREDEBL
```bash
cd credebl
cp .env.example .env && nano .env
VPS_IP=$(curl -s ifconfig.me) && sed -i "s/YOUR_VPS_IP/$VPS_IP/g" .env
docker compose pull && docker compose up -d
bash ../scripts/health-check.sh
```

### INJI
```bash
export CERTIFY_KEYSTORE_PASSWORD=$(openssl rand -hex 16)
bash scripts/generate-inji-certs.sh

cd inji
cp .env.example .env && nano .env
VPS_IP=$(curl -s ifconfig.me)
sed -i "s/YOUR_VPS_IP/$VPS_IP/g" .env
sed -i "s/YOUR_VPS_IP/$VPS_IP/g" config/mimoto/mimoto-issuers-config.json
sed -i "s/YOUR_VPS_IP/$VPS_IP/g" config/mimoto/mimoto-trusted-verifiers.json

# INJI images are AMD64 only
export DOCKER_DEFAULT_PLATFORM=linux/amd64
docker compose pull

# Start in order (each service needs its dependency healthy first)
docker compose up -d postgres redis
sleep 20
docker compose up -d esignet
sleep 90
docker compose up -d inji-certify certify-nginx
sleep 120
docker compose up -d mimoto inji-web mailpit
bash ../scripts/health-check-inji.sh
```

## Important вЂ” before first deploy

**CREDEBL**: The `seed` service requires the full upstream-style `credebl-master-table.json` in `credebl/config/`. Do not reduce it to only ledger rows or `org_agents_type` entries вЂ” the `credebl-seed` container also expects keys such as `orgRoleData`, `agentTypeData`, `platformAdminData`, and `ledgerConfig`.

**INJI**: Always set `export DOCKER_DEFAULT_PLATFORM=linux/amd64` вЂ” images have no ARM64 build.  
Run `generate-inji-certs.sh` before first deploy to create the required keystore.
